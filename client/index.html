    Introduction

    Welcome to the RELIABLE-DOCS API! You can use our API to access user , role, document API endpoints, which can get information on various user, document, and roles in our database.

    Authentication

    • All routes are protected and needs a valid token except the login and signup routes. To get a token, the user signs up or logs in(If the user has signed up previously). A token is included in the response from the signup/login route, this token expires after 24hours. With a valid token a user can create/update/delete or search for documents, the request must contain the valid token on the header of the request in the following format.

    Authorization: user_token

    You must replace user_token with your personal token.

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files...just save them to the `includes` folder and add them to the top of your `index.md`'s frontmatter. Files are included in the order listed.

    The Reliable-Docs API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- The server did not understand the request.
    401 Unauthorized -- Invalid parameter supplied.
    403 Forbidden -- Access is forbidden to the requested page.
    404 Not Found -- The server can not find the requested page.
    412 Precondition Failed - The pre condition given in the request evaluated to false by the server.
    500 Internal Server Error -- The request was not completed. The server met an unexpected condition.
    503 Service Unavailable -- The request was not completed. The server is temporarily overloading or down.
